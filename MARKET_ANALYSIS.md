# 모바일 앱 네비게이션 시장 분석

## 조사 목적
사진 편집 및 AI 얼굴 교체 앱의 네비게이션 패턴을 분석하여 "머리보존 AI" 앱에 적용할 최적의 네비게이션 구조를 설계한다.

## 1. 하단 탭바 (Bottom Navigation Bar)

### 업계 표준 패턴
하단 탭바는 모바일 앱에서 가장 보편적인 네비게이션 패턴으로, 3-5개의 주요 섹션에 빠르게 접근할 수 있도록 설계된다.

### 일반적인 구성 (3-5개 탭)

| 탭 위치 | 기능 | 목적 | 사용 예시 |
|---------|------|------|-----------|
| 1번 (왼쪽) | 홈 / 피드 | 메인 콘텐츠 접근 | Instagram (홈), Secondhand (홈) |
| 2번 | 탐색 / 검색 | 콘텐츠 발견 | Instagram (검색), Spotify (검색) |
| 3번 (중앙) | 주요 액션 | 핵심 기능 실행 (강조) | Instagram (+), Secondhand (+) |
| 4번 | 활동 / 메시지 | 알림, 소통 | Instagram (하트), Secondhand (메시지) |
| 5번 (오른쪽) | 프로필 / 설정 | 개인 정보, 설정 | Instagram (프로필), Spotify (라이브러리) |

### 핵심 원칙
1. **엄지 도달 범위**: 하단 배치로 한 손 조작 최적화
2. **명확한 아이콘**: 텍스트 없이도 기능 인지 가능
3. **활성 상태 표시**: 현재 위치를 색상/굵기로 명확히 표시
4. **중앙 강조**: 주요 액션은 중앙에 크게 배치 (선택적)

### 사진 편집 앱 특화 패턴
- **홈**: 최근 작업물, 새 프로젝트 시작
- **갤러리/라이브러리**: 저장된 작업물 탐색
- **편집 (중앙)**: 새 사진 편집 시작 (주요 액션)
- **히스토리**: 편집 기록, 버전 관리
- **설정/프로필**: 구독 관리, 앱 설정

## 2. 상단 네비게이션 바 (Top Navigation Bar)

### 업계 표준 구조
상단 바는 컨텍스트 정보와 액션을 제공하며, 일반적으로 3개 영역으로 구성된다.

| 영역 | 위치 | 기능 | 목적 |
|------|------|------|------|
| 왼쪽 | Leading | 뒤로가기, 닫기, 메뉴 | 네비게이션 제어 |
| 중앙 | Center | 화면 제목, 로고 | 현재 위치 표시 |
| 오른쪽 | Trailing | 액션 버튼 (저장, 공유, 더보기) | 주요 작업 실행 |

### 화면별 상단 바 패턴

#### 홈 화면
- **왼쪽**: 로고 또는 앱 이름
- **중앙**: (비어있음 또는 검색바)
- **오른쪽**: 설정, 알림

#### 편집 화면
- **왼쪽**: 닫기 (X) 또는 뒤로가기 (<)
- **중앙**: "편집" 또는 파일명
- **오른쪽**: 완료, 저장, 공유

#### 상세 화면
- **왼쪽**: 뒤로가기 (<)
- **중앙**: 화면 제목
- **오른쪽**: 더보기 (⋮), 공유, 삭제

### 아이콘 사용 원칙
1. **뒤로가기**: < (왼쪽 화살표) 또는 "뒤로"
2. **닫기**: × (X 아이콘)
3. **더보기**: ⋮ (세로 점 3개) 또는 ⋯ (가로 점 3개)
4. **공유**: 공유 아이콘 (iOS 스타일 또는 Android 스타일)
5. **저장**: 체크 아이콘 (✓) 또는 "저장" 텍스트

## 3. 사진 편집 앱 UX 핵심 원칙

### Polarr 앱 재설계 사례 (UX 연구)
전문가용 사진 편집 앱 Polarr의 UX 개선 프로젝트에서 도출된 3가지 핵심 목표:

1. **편집 프로세스 단순화**
   - 복잡한 기능을 초보자도 쉽게 사용할 수 있도록
   - 단계별 가이드 제공

2. **편집 속도 향상**
   - 자주 사용하는 기능에 빠른 접근
   - 불필요한 단계 제거

3. **익숙한 프레임워크 사용**
   - 다른 사진 편집 앱과 유사한 UI 패턴
   - 학습 곡선 최소화

### 필수 UX 요소

#### 1. 명확한 CTA (Call-to-Action)
- "저장", "삭제", "공유" 등 구체적인 동사 사용
- 모호한 "확인", "제출" 금지
- 버튼 위치: 엄지 도달 범위 내 (하단 또는 상단 오른쪽)

#### 2. 배터리 소모 최소화
- 어두운 색상 사용 (다크 모드)
- 불필요한 배경 서비스 제한
- 위치 추적 최소화

#### 3. 대역폭 고려
- 재사용 가능한 요소 활용
- 추가 다운로드 시 사용자 동의 요청

## 4. 2024-2026 모바일 앱 UX 트렌드

### 네비게이션 트렌드
1. **하단 중심 디자인**: 탭바, 버튼을 하단에 배치하여 접근성 향상
2. **제스처 기반 네비게이션**: 스와이프, 드래그로 화면 전환
3. **카드 기반 UI**: 콘텐츠를 카드 형태로 배치하여 스크롤 최소화
4. **탭 바 강조**: 명확한 아이콘과 레이블로 현재 위치 표시

### 버튼 디자인 트렌드
1. **둥근 버튼**: 날카로운 모서리 대신 둥근 모서리 (rounded-full)
2. **컬러풀한 CTA**: 흰색 배경에 대비되는 밝은 색상
3. **큰 터치 영역**: 최소 44x44pt (iOS), 48x48dp (Android)

### 카메라 통합
- 사진 편집 앱뿐만 아니라 다양한 앱에서 카메라 기능 통합
- 바코드 스캔, QR 코드, 신용카드 스캔 등

## 5. "머리보존 AI" 앱 적용 방안

### 하단 탭바 구성 (4개 탭)

| 탭 | 아이콘 | 기능 | 목적 |
|-----|--------|------|------|
| 홈 | 집 아이콘 | 메인 화면, 새 프로젝트 시작 | 핵심 기능 접근 |
| 히스토리 | 시계/폴더 아이콘 | 과거 작업물 보기 | 재사용, 관리 |
| 설정 | 톱니바퀴 아이콘 | 구독, 앱 설정 | 개인화, 관리 |
| 프로필 | 사람 아이콘 | 계정 정보, 로그아웃 | 사용자 관리 |

**중앙 강조 버튼 제거 이유**:
- 홈 화면에 이미 큰 "새 프로젝트 시작" 버튼 존재
- 4개 탭으로 균형잡힌 레이아웃 유지

### 상단 네비게이션 바 패턴

#### 홈 화면
- **왼쪽**: "머리보존" 텍스트 로고
- **중앙**: (비어있음)
- **오른쪽**: 알림 아이콘, 설정 아이콘

#### 사진 선택 화면
- **왼쪽**: 뒤로가기 (<)
- **중앙**: "사진 선택"
- **오른쪽**: (비어있음)

#### 스타일 선택 화면
- **왼쪽**: 뒤로가기 (<)
- **중앙**: "스타일 선택"
- **오른쪽**: 도움말 아이콘 (?)

#### 얼굴 선택 화면
- **왼쪽**: 뒤로가기 (<)
- **중앙**: "얼굴 선택"
- **오른쪽**: 필터 아이콘

#### 결과 확인 화면
- **왼쪽**: 뒤로가기 (<)
- **중앙**: "결과 확인"
- **오른쪽**: 공유 아이콘, 더보기 (⋮)

### 아이콘 시스템
**이모지 사용 금지** - 대신 다음 사용:
1. **SVG 아이콘**: react-native-vector-icons 또는 expo-vector-icons
2. **커스텀 아이콘**: Figma에서 디자인 후 SVG로 export
3. **시스템 아이콘**: iOS SF Symbols, Android Material Icons

### 권장 아이콘 라이브러리
- **Ionicons**: 범용 아이콘 세트
- **MaterialIcons**: Android 스타일
- **Feather**: 미니멀한 라인 아이콘

## 6. 경쟁사 분석

### Instagram
- **하단 탭**: 홈, 검색, +, 하트, 프로필 (5개)
- **상단 바**: 로고 (왼쪽), DM 아이콘 (오른쪽)

### VSCO (사진 편집 앱)
- **하단 탭**: 탐색, 편집, 프로필 (3개)
- **상단 바**: 뒤로가기, 제목, 저장/공유

### Snapseed (Google 사진 편집)
- **하단 탭**: 없음 (단일 화면 중심)
- **상단 바**: 뒤로가기, 제목, 더보기
- **하단 액션 바**: 편집 도구 가로 스크롤

### FaceApp (AI 얼굴 편집)
- **하단 탭**: 홈, 갤러리, 프로필 (3개)
- **상단 바**: 로고, 프리미엄 버튼
- **편집 화면**: 하단에 필터 옵션 가로 스크롤

## 7. 결론 및 권장사항

### 핵심 원칙
1. **일관성**: 모든 화면에서 동일한 네비게이션 패턴 유지
2. **접근성**: 하단 중심 배치로 한 손 조작 최적화
3. **명확성**: 아이콘과 레이블로 기능 명확히 전달
4. **단순성**: 3-4개 주요 섹션으로 제한

### 구현 우선순위
1. **Phase 1**: 하단 탭바 4개 (홈, 히스토리, 설정, 프로필)
2. **Phase 2**: 상단 네비게이션 바 (뒤로가기, 제목, 액션)
3. **Phase 3**: 아이콘 시스템 (SVG 아이콘으로 이모지 대체)
4. **Phase 4**: 제스처 네비게이션 (스와이프 뒤로가기)

### 차별화 포인트
- **투명성 강조**: "머리는 절대 변형되지 않았습니다" 메시지를 상단 바에 표시
- **빠른 재작업**: 히스토리에서 이전 작업물 불러와 다른 얼굴로 재시도
- **비교 모드**: 결과 화면에서 좌우 슬라이더로 실시간 비교
