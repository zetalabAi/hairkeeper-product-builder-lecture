# 머리보존 AI - 모바일 앱 디자인 계획

## 디자인 원칙

### 플랫폼 기준
- **Apple Human Interface Guidelines (HIG) 준수**
- iOS 네이티브 앱과 동일한 느낌
- 모바일 세로 방향 (9:16) 최적화
- 한 손 조작 가능한 UI

### UX 철학
- B612 셀카 앱과 유사한 직관적 감성
- 텍스트 입력 없음 (선택만 가능)
- 설명 최소화, 직관 최대화
- 빠른 처리 (2.5~6초 목표)

## 화면 구조

### 1. 온보딩 화면 (Onboarding)
**목적**: 앱의 핵심 가치 전달 및 첫인상 형성

**콘텐츠**:
- 슬라이드 1: "실제 시술, 안전한 공유" - 헤어 보존 개념 소개
- 슬라이드 2: "얼굴만 바꿔요" - Before/After 예시
- 슬라이드 3: "3초 만에 완성" - 간편함 강조
- 슬라이드 4: 소셜 로그인 버튼 (Google, Naver, Kakao)

**주요 기능**:
- 수평 스와이프로 슬라이드 전환
- 하단 인디케이터 (점 3개)
- "건너뛰기" 버튼 (우상단)
- 마지막 슬라이드에서 "시작하기" 버튼

### 2. 로그인 화면 (Login)
**목적**: 간편한 소셜 로그인

**콘텐츠**:
- 앱 로고 (중앙 상단)
- 앱 이름 "머리보존 AI"
- 태그라인 "AI지만, 머리는 진짜."
- 소셜 로그인 버튼 3개 (세로 배치)
  - Google 로그인
  - Naver 로그인
  - Kakao 로그인

**주요 기능**:
- 각 버튼은 해당 플랫폼 브랜드 컬러 사용
- 버튼 탭 시 OAuth 플로우 시작

### 3. 홈 화면 (Home)
**목적**: 메인 작업 시작점 및 히스토리 확인

**콘텐츠**:
- 상단: 앱 로고 + 설정 아이콘 (우상단)
- 중앙: 큰 "사진 선택" 버튼 (카메라 아이콘)
  - "촬영하기" 옵션
  - "앨범에서 선택" 옵션
- 하단: 최근 작업 히스토리 (가로 스크롤 썸네일)
  - 각 썸네일 탭 시 결과 화면으로 이동

**주요 기능**:
- 사진 선택 시 권한 요청 (카메라/앨범)
- 히스토리는 최대 20개까지 표시

### 4. 사진 편집 화면 (Edit)
**목적**: 얼굴 교체 옵션 선택

**콘텐츠**:
- 상단: 선택한 원본 사진 (전체 화면)
- 하단 패널 (슬라이드업):
  - Step 1: 국적 선택 (한국/일본)
  - Step 2: 성별 선택 (남성/여성)
  - Step 3: 스타일 선택
    - 여성: 청순 / 귀여움 / 아름다움 / 도도 / 섹시
    - 남성: 늠름 / 섹시 / 남성적 / 강한 인상
  - "다음" 버튼 (하단 고정)

**주요 기능**:
- 각 단계는 칩(Chip) 형태로 선택
- 선택 시 하이라이트 효과
- 모든 선택 완료 시 "다음" 버튼 활성화

### 5. 얼굴 선택 화면 (Face Selection)
**목적**: AI 생성 얼굴 후보 중 선택

**콘텐츠**:
- 상단: "추천 얼굴" 제목
- 그리드 레이아웃 (2열)
  - 각 셀: 얼굴 썸네일 + "유사도 %" 라벨
  - 추천 순위 1위는 "추천" 배지 표시
- 하단: "적용하기" 버튼

**주요 기능**:
- 얼굴 탭 시 선택 표시 (테두리 강조)
- 유사도는 서버에서 계산된 값 표시
- 최대 6개 후보 표시

### 6. 결과 확인 화면 (Result Preview)
**목적**: Before/After 비교 및 저장

**콘텐츠**:
- 전체 화면: 합성된 결과 이미지
- Before/After 슬라이더 (수평 드래그)
- 하단 버튼:
  - "다시 선택" (얼굴 재선택)
  - "저장하기" (메인 액션)

**주요 기능**:
- 슬라이더 드래그로 원본/결과 비교
- 무료 사용자: 워터마크 표시 + 광고
- 유료 사용자: 고화질 저장

### 7. 저장 완료 화면 (Save Complete)
**목적**: 저장 확인 및 다음 액션 유도

**콘텐츠**:
- 중앙: 체크 아이콘 + "저장 완료!" 메시지
- 저장된 이미지 썸네일
- 하단 버튼:
  - "공유하기" (시스템 공유 시트)
  - "새로 만들기" (홈으로 이동)

**주요 기능**:
- 무료 사용자: 광고 표시 (전면 광고)
- 공유 시 시스템 공유 기능 사용

### 8. 설정 화면 (Settings)
**목적**: 계정 관리 및 구독 설정

**콘텐츠**:
- 프로필 섹션:
  - 사용자 이름
  - 이메일
  - 구독 상태 (무료/프리미엄)
- 구독 섹션:
  - "프리미엄 구독" 버튼 (무료 사용자만)
  - 구독 혜택 안내
  - 가격: 월 11,900원
- 기타:
  - 이용약관
  - 개인정보처리방침
  - 로그아웃

**주요 기능**:
- 구독 버튼 탭 시 인앱 결제 시트
- 로그아웃 시 확인 다이얼로그

## 주요 사용자 플로우

### 플로우 1: 첫 사용자 (온보딩 → 로그인 → 첫 작업)
1. 앱 실행 → 온보딩 화면 (3개 슬라이드)
2. "시작하기" 탭 → 로그인 화면
3. 소셜 로그인 선택 (Google/Naver/Kakao) → OAuth 인증
4. 로그인 완료 → 홈 화면
5. "사진 선택" 탭 → 권한 요청 → 사진 선택
6. 사진 편집 화면 → 국적/성별/스타일 선택 → "다음"
7. 얼굴 선택 화면 → 얼굴 선택 → "적용하기"
8. 로딩 (2.5~6초) → 결과 확인 화면
9. Before/After 슬라이더로 확인 → "저장하기"
10. 광고 표시 (무료 사용자) → 저장 완료 화면
11. "공유하기" 또는 "새로 만들기"

### 플로우 2: 재방문 사용자 (빠른 작업)
1. 앱 실행 → 홈 화면 (로그인 상태 유지)
2. "사진 선택" → 사진 선택
3. 국적/성별/스타일 선택 (이전 선택 기억)
4. 얼굴 선택 → 적용
5. 결과 확인 → 저장
6. 완료

### 플로우 3: 프리미엄 구독
1. 홈 화면 → 설정 아이콘 탭
2. 설정 화면 → "프리미엄 구독" 버튼 탭
3. 인앱 결제 시트 → 결제 정보 입력
4. 결제 완료 → 구독 상태 업데이트
5. 설정 화면으로 복귀 (프리미엄 배지 표시)

## 인터랙션 디자인

### 버튼 피드백
- **Primary 버튼**: Scale 0.97 + 가벼운 햅틱
- **리스트 아이템**: Opacity 0.7
- **아이콘 버튼**: Opacity 0.6

### 애니메이션
- 화면 전환: 250ms fade + slide
- 버튼 탭: 80ms scale
- 로딩: Spinner (1초 이상 소요 시)

### 햅틱
- 버튼 탭: Light impact
- 선택 완료: Medium impact
- 저장 완료: Success notification
- 오류 발생: Error notification

## 기술 제약사항

### 온디바이스 처리
- 얼굴/헤어 세그멘테이션
- 얼굴 후보 추천 (유사도 계산)
- UI 렌더링

### 서버 처리
- AI 얼굴 합성 (Nano Banana 또는 최고 성능 모델)
- 얼굴 풀(Pool) 관리
- 사용자 데이터 동기화
- 결제 처리

### 성능 목표
- 합성 처리: 2.5~6초 (최악 10초)
- 앱 시작: 1초 이내
- 화면 전환: 250ms 이내

## 접근성

- 다크모드 완벽 지원
- 최소 터치 영역: 44x44pt (iOS HIG)
- 색각 이상자 고려 (컬러 + 아이콘 병행)
- VoiceOver/TalkBack 지원 (접근성 라벨)

## 워터마크 디자인 (무료 사용자)

- 위치: 우하단
- 크기: 이미지 너비의 15%
- 텍스트: "머리보존 AI" + 로고
- 투명도: 50%
- 색상: 흰색 (배경에 따라 자동 조정)
